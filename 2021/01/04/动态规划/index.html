动态规划

发表于2021-01-04|更新于2021-01-04|数据结构与算法
|字数总计:1.5k|阅读时长:5分钟|阅读量:

适用动态规划的问题

计数问题

  • 有多少种方式走到右下角
  • 有多少种方法选出K个数使和为sum

求最大值和最小值

  • 从左上角到右下角路径的最大数字和
  • 最长上升子序列长度

求存在性

  • 取石子问题，先手是否必胜
  • 能不能选出K个数使得和为sum

解题框架

  1. 确定状态：子问题和最后一步。找到最后一步将一个大问题转化为一个小问题
  2. 根据子问题的转化过程，确定状态转移方程。定义状态转移方程，一般用数组表示，有几个状态就对应着几维数组
  3. 确定初始条件和边界情况。初始条件一般是我们转移方程求不出来的量，边界情况一般要考虑数组索引是否小于是否下标越界等。
  4. 弄清楚计算顺序。一般是从小到大，从上左到右的顺序

例题

LeetCode 70 爬楼梯

思路分析

  1. 通过读题，我们发现问题是”有多少种不同的方法可以爬到楼顶”，很显然这是动态规划中的第一类问题计数问题
  2. 确定状态,找到最后一步:一共n阶楼梯，如果我们爬到了n-ak阶再爬ak阶是不是就到达了楼顶了。因此我们就将求爬n阶梯的方法转化为了求爬n-ak阶的方法，因为只有阶梯数是在变得因此本题的状态就是阶梯数量
  3. 确定转移方程:题目中说我每次可以爬1阶或2阶，因此ak就等于1或2，因为本题只有一个状态，因此只需定义一个一维数组即可，因此我们爬n阶楼梯的方法数就是爬n-1阶楼梯的方法数与爬n-2阶楼梯方法数之和。dp[n] = dp[n-1] + dp[n-2]
  4. 确定初始条件: 爬一阶与不爬的方法都是1种因此dp[0] = dp[1] = 1

LeetCode746 使用最小花费爬楼梯

思路分析

  1. 题目要求”找出达到楼层顶部的最低花费”很显然这是动态规划的第二类问题求最值问题
  2. 找状态: 爬n阶梯需要的最小花费等于Min(爬n-1阶梯的最小花费加爬到顶的花费，爬n-2阶梯的最小花费加爬到顶的花费)
  3. cost[2] = 20 代表爬到第2阶的花费体力为20(下标从 0 开始)
  4. 状态转移方程 dp[n] = min(dp[n-1] + cost[n-1],dp[n-2]+cost[n-2])
  5. 初始条件:dp[0] =dp[1]= 0 初始站得位置不需要花费体力。

LeetCode 62 不同路径

思路分析

  1. 题目中描述 机器人初始时站在左上角 问到右下角有多少条不同路径，很显然是我们动态规划的典型题型
  2. 找状态,看最后一步:如果我想到到达坐标(m-1.n-1)无非两种方式能到达从(m-2,n-1)向下走一步或(m-1,n-2)向右走一步，因此我们可以将机器人走到m-1,n-1位置的问题转化为机器人走到m-2,n-1的路径与m-1,n-2的路径这个子问题
  3. 由上述分析不难推出状态转移方程 dp[i] [j] = dp[i-1] [j] + dp[i] [j-1]
  4. 确定初始条件 ；因为机器人只能向右或向下走，初始时机器人是站在左上角的因此机器人在第一行或第一列的移动方式只有一种
